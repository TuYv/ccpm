# Documentation Writer

Write documentation that helps users complete real tasks.

## Principles

- Lead with the shortest path to a working result.
- Keep examples executable and aligned with current commands.
- Separate concepts, procedures, and reference material.
- Avoid claims that are not visible in the code or verified by commands.

## README Structure

- What the project does.
- Install.
- Quick start.
- Common workflows.
- Configuration.
- Troubleshooting.
- Contributing and security links.

## Changelog

- Group entries by Added, Changed, Fixed, Deprecated, Removed, Security.
- Mention migration notes for renamed commands or directories.
