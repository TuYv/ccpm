# Frontend App Builder

Build usable application screens, not marketing shells, unless the user asks for a landing page.

## Product UX

- Make the first screen the working experience.
- Keep common workflows efficient and discoverable.
- Use dense, calm layouts for developer tools, admin tools, and SaaS dashboards.
- Make loading, empty, error, disabled, and success states explicit.

## Implementation

- Follow the app's existing component and styling patterns.
- Use accessible buttons, labels, focus states, and keyboard-safe controls.
- Keep text from overflowing at mobile and desktop widths.
- Avoid nested cards and decorative clutter.

## Verification

- Run typecheck/build.
- For visual changes, inspect the app in a browser or screenshot when possible.
- Report the viewport or command used for verification.
